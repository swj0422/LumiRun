# 逐光成长系统（LumiRun）设计文档

## 1. 系统整体架构说明

### 1.1 前后端架构

- **前端**：Vue 3 + TypeScript + Vite
  - **请求优化**：
    - 防抖/节流：批量导入成长值时的按钮防重复点击、搜索框输入节流
    - 异步请求管理：使用 axios 拦截器统一处理请求和响应
    - WebSocket 客户端：实现实时事件推送，支持重连机制
  - **性能优化**：
    - 分页加载：列表数据每页显示20条，翻页再加载下一页
    - 按需加载：登录后只刷新当前需要展示的页面
    - 局部更新：编辑操作后只刷新当前行，不刷新整个页面
    - 缓存机制：使用localStorage存储菜单、权限、基础配置等数据
- **后端**：FastAPI + SQLAlchemy (async) + MySQL
  - **性能优化**：
    - 异步处理：使用FastAPI的BackgroundTasks处理批量导入等耗时操作
    - 数据库优化：合理使用索引，优化查询语句，实现批量查询减少N+1问题
    - 缓存机制：使用Redis缓存高频查询接口，如排行榜、学员列表、成长值记录
    - 性能监控：添加请求时间统计和日志记录，便于性能分析
- **通信**：
  - RESTful API：常规业务操作
  - WebSocket：实时事件推送（库存变动、订单审核、心愿提醒等）
- **缓存**：Redis 缓存（高频查询接口，如排行榜、学员列表、成长值记录），设置 5 分钟过期时间
  - **缓存键设计**：
    - 排行榜：`leaderboard:v2:{class_id}:{teacher_id}:{order_by}:{limit}:{status}`
    - 成长值：`growth_score:{class_student_id}`
    - 成长值流水：`growth_logs:{teacher_id}:{class_id}:{student_name}:{filters}:{skip}:{limit}`
    - 成长值历史：`growth_history:{teacher_id}:{student_name}:{filters}:{skip}:{limit}`
    - 班级排行榜：`class_leaderboard:{teacher_id}`
    - 学员列表：`students:teacher:{teacher_id}:class:{class_id or 'all'}:keyword:{keyword or 'none'}:status:{status or 'all'}:sort:{sort or 'none'}:order:{order}:school:{school_name or 'none'}:skip:{skip}:limit:{limit}`
    - 历史学员：`students:history:teacher:{teacher_id}:class:{class_id or 'all'}:keyword:{keyword or 'none'}:skip:{skip}:limit:{limit}`
    - **版本号说明**：缓存键添加版本号（如 v2），确保代码修改后缓存会更新
  - **缓存策略**：
    - 主动更新：数据变更时主动清除相关缓存
    - 过期时间：根据数据更新频率设置不同的过期时间（2-5分钟）
    - 缓存命中监控：添加缓存命中日志，便于性能分析
    - 批量查询：实现批量获取学员成长值、标签等数据，减少N+1查询问题
- **中间件**：CORS、性能监控、错误处理
- **静态文件服务**：上传文件和前端静态资源
- **API路由**：/api/v1 前缀
- **WebSocket 端点**：/api/ws
- **主要API端点**：
  - `/api/v1/auth`：认证相关
  - `/api/v1/users`：用户管理
  - `/api/v1/role`：角色权限管理
  - `/api/v1/classes`：班级管理
  - `/api/v1/students`：学员绑定（扫码绑定）
  - `/api/v1/class_student`：班级学员管理（导师添加的学员）
  - `/api/v1/growth`：成长值管理
  - `/api/v1/gifts`：奖励管理
  - `/api/v1/orders`：奖励兑换
  - `/api/v1/leaderboard`：排行榜
  - `/api/v1/logs`：系统日志
  - `/api/v1/system-logs`：系统操作日志
  - `/api/v1/student-operation-logs`：学员操作日志
  - `/api/v1/messages`：消息通知
  - `/api/v1/export`：数据导出
  - `/api/v1/wishes`：学员心愿
  - `/api/v1/tags`：标签管理
  - `/api/v1/admin`：管理员功能
- **健康检查**：/health 端点

### 1.2 角色体系

| 角色           | 描述     | 权限范围                 |
| ------------ | ------ | -------------------- |
| super\_admin | 系统最高权限 | 所有功能，包括用户管理、系统配置     |
| admin        | 系统管理权限 | 用户管理、系统监控            |
| teacher      | 班级管理权限 | 创建班级、管理学员、记录成长值、审核兑换 |
| student      | 基础使用权限 | 查看成长值、兑换礼品、绑定班级、发布意见 |
| class\_assistant | 班级助理权限 | 查看被授权班级的成长值、排行、记录、发布意见等   |

**多身份系统说明**：
- 用户可以同时拥有多个身份（如既是学员又是助理）
- 用户自主注册时只能是导师或学员身份
- 助理身份只能通过导师授权获得
- 用户切换身份后，需要重新登录以获取对应身份的 token
- **身份切换后的成长记录过滤**：
  - 助理身份：查看自己操作的成长记录（`operator_id` = 当前用户）
  - 学员身份：查看自己被加减的成长记录（`class_student_id` = 当前用户的班级学员ID）
- **身份参数支持**：成长记录API支持`identity`参数，允许用户指定以助理或学员身份查看记录

## 3. 完整业务流程逻辑

### 3.1 注册登录流程

1. **注册**：用户填写信息（邮箱、用户名、真实姓名） → 选择角色 → 提交注册 → 系统验证 → 发送验证码 → 验证通过 → 提交审核（导师需要管理员审核，学员无需审核）
2. **登录**：用户输入用户名/邮箱和密码 → 系统验证 → 生成token → 登录成功
   - **验证码机制**：连续登录错误3次后，系统要求输入图形验证码
   - **账号锁定**：连续登录错误达到阈值后，账号会被暂时锁定
3. **忘记密码**：用户填写注册邮箱 → 系统发送重置链接 → 用户点击链接 → 输入新密码 → 密码重置成功
4. **令牌刷新**：当访问令牌过期时，用户可以使用刷新令牌获取新的访问令牌，无需重新登录

### 3.2 导师创建班级、生成邀请码流程

1. 导师登录 → 进入班级管理 → 点击"创建班级" → 填写班级信息 → 提交 → 系统生成唯一邀请码 → 导师获取邀请码

### 3.3 学员扫码绑定班级流程

1. 学员扫描邀请码 → 系统验证邀请码 → 填写个人信息 → 提交 → 系统创建绑定关系 → 绑定成功

### 3.4 导师给学员加减成长值流程

1. **单个记录**：导师登录 → 进入成长管理 → 点击"记录" → 选择学员 → 输入成长值变动 → 填写原因 → 提交 → 系统更新成长值 → 记录成长日志（操作类型为"添加"，不论成长值正负）→ 记录成长操作日志
2. **批量导入**：导师登录 → 进入成长管理 → 点击"批量导入" → 选择班级 → 下载模板 → 填写Excel文件（包含学员姓名、成长值变动、原因）→ 上传文件 → 系统解析并提交异步任务 → 显示任务提交成功提示 → 后台异步处理成长值记录 → 前端刷新查看导入结果（操作类型为"添加"，不论成长值正负）→ 记录成长操作日志
3. **编辑记录**：导师登录 → 进入成长管理 → 点击记录的"编辑"按钮 → 修改成长值或原因 → 提交 → 系统更新记录 → 记录成长操作日志
4. **删除记录**：导师登录 → 进入成长管理 → 点击记录的"编辑"按钮 → 点击"删除"按钮 → 填写删除原因 → 提交 → 系统添加一条负的成长值记录来表示删除（操作类型为"删除"，记录数值为删除记录的负值，保持操作可溯源）→ 记录成长操作日志

### 3.5 学员管理与学员日志流程

1. **学员管理**：导师登录 → 进入学员管理 → 查看学员列表 → 支持按学校、届、班级筛选 → 支持搜索学员姓名（缓存有效期5分钟）
2. **添加学员**：导师登录 → 进入学员管理 → 点击"添加学员" → 填写学员信息 → 提交 → 系统创建学员记录 → 清除相关缓存 → 记录操作日志
3. **批量添加学员**：导师登录 → 进入学员管理 → 点击"批量添加" → 上传Excel文件 → 系统解析并创建学员记录 → 清除相关缓存 → 记录操作日志
4. **停用学员**：导师登录 → 进入学员管理 → 点击学员的"更多"按钮 → 点击"停用" → 填写停用原因 → 提交 → 系统标记学员为已删除 → 清除相关缓存 → 记录操作日志
5. **解绑学员**：导师登录 → 进入学员管理 → 点击学员的"更多"按钮 → 点击"解绑" → 填写解绑原因 → 提交 → 系统标记学员为已删除 → 清除相关缓存 → 记录操作日志
6. **恢复学员**：导师登录 → 进入学员管理 → 点击"历史学员"页签 → 找到要恢复的学员 → 点击"恢复"按钮 → 系统标记学员为未删除 → 清除相关缓存 → 记录操作日志
7. **学员日志**：导师登录 → 进入学员管理 → 点击"学员日志"页签 → 查看学员相关操作记录 → 支持按操作类型筛选（添加/解绑/标签/备注/停用/恢复）
8. **历史学员**：导师登录 → 进入学员管理 → 点击"历史学员"页签 → 查看所有已删除的学员 → 支持搜索学员姓名 → 支持恢复学员（缓存有效期5分钟）

### 3.6 成长值计算逻辑

1. **总成长值**：只包括通过成长管理增减的记录，不包括兑换记录
2. **可用成长值**：包括所有成长记录的增减，包括兑换记录的扣除和删除操作的负记录
3. **成长记录显示**：成长记录页面只显示通过成长管理增减的记录，不显示兑换记录
4. **排行榜计算**：排行榜同时显示总成长值和可用成长值
5. **删除操作处理**：删除成长值记录时，系统会添加一条负的成长值记录来表示删除，保持操作可溯源，不直接删除原记录
6. **成长值记录存储**：成长值记录存储在growth表中，成长操作日志存储在growth\_operation\_log表中，确保所有操作都有详细的日志记录

### 3.6 礼品管理与兑换流程

1. **礼品管理**：
   - 导师登录 → 进入礼品管理 → 点击"创建礼品" → 填写礼品信息（名称、描述、价格、库存）→ 上传图片 → 提交 → 系统创建礼品和库存记录
   - 导师可以编辑、删除礼品，调整库存和状态
   - 导师可以将礼品开放给特定班级
2. **礼品兑换**：学员登录 → 进入礼品页面 → 选择礼品 → 确认兑换 → 系统扣减积分和库存 → 生成订单
3. **订单审核**：导师登录 → 进入订单管理 → 审核订单 → 生成核销码
4. **线下核销**：导师/管理员扫描核销码 → 系统验证 → 核销成功
5. **库存管理**：系统提供库存预警功能，当库存低于阈值时提醒导师

### 3.7 心愿提醒、库存变动、消息推送流程

1. **心愿提醒**：学员设置心愿 → 系统监测库存 → 库存充足时通过 WebSocket 实时提醒学员
2. **库存变动**：导师修改礼品库存 → 系统更新库存 → 通过 WebSocket 实时推送库存变动信息
3. **消息推送**：系统根据事件（审核通过、兑换成功等）通过 WebSocket 向用户推送实时消息
4. **WebSocket 功能**：
   - 连接管理：支持用户与班级的多对多连接
   - 实时事件：库存变动、订单审核、心愿提醒
   - 心跳机制：保持连接活跃
   - 重连机制：网络中断后自动重连

### 3.8 班级开放/关闭后的权限控制流程

1. **开放**：班级状态为开放 → 学员可绑定 → 导师可管理
2. **关闭**：班级状态为关闭 → 学员不可绑定 → 导师仍可管理历史数据

### 3.9 角色管理流程

1. **超级管理员/管理员登录** → 进入角色管理页面 → 点击"创建角色" → 填写角色信息（名称、描述） → 提交 → 系统创建角色
2. **编辑角色**：进入角色管理页面 → 选择角色 → 点击"编辑" → 修改角色信息 → 保存
3. **删除角色**：进入角色管理页面 → 选择角色 → 点击"删除" → 确认删除 → 系统删除角色

### 3.10 导师授权班级助理流程

1. 导师登录 → 进入班级管理 → 选择班级 → 点击"班级助理" → 添加助理 → 输入助理邮箱 → 提交 → 系统创建助理授权关系
2. 助理收到授权通知（邮件/消息） → 助理登录系统 → 可以查看被授权班级的数据
3. 助理可以切换身份：以学员身份绑定班级，或以助理身份查看授权班级数据

### 3.11 多身份切换流程

1. 用户拥有多个身份 → 登录系统 → 选择当前身份（导师/学员/助理）
2. **助理切换为学员身份**：助理点击"绑定班级" → 系统清除当前 token → 跳转到登录页 → 用户用学员身份登录 → 绑定班级
3. **助理查看授权班级**：助理选择助理身份 → 系统验证助理权限 → 显示被授权的班级数据

### 3.10 权限配置流程

1. **超级管理员/管理员登录** → 进入权限管理页面 → 选择角色（系统内置角色或自定义角色）→ 配置权限和页面访问控制 → 保存配置
2. **配置生效**：系统更新角色权限配置 → 重新加载用户权限 → 权限变更立即生效

## 5. 完整数据库表结构设计

### 5.1 用户表（sys\_user）

| 字段名                      | 类型           | 主键 | 外键           | 是否为空 | 用途说明           |
| ------------------------ | ------------ | -- | ------------ | ---- | -------------- |
| id                       | INTEGER      | 是  | 否            | 否    | 用户ID           |
| email                    | VARCHAR(100) | 否  | 否            | 否    | 邮箱（用于找回密码）     |

## 7. 权限判断逻辑

### 7.1 不同角色能看到什么、不能看到什么

- **超级管理员**：所有功能和数据，包括配置其他角色的权限和页面
- **管理员**：用户管理、系统日志、配置导师和学生的权限和页面
- **导师**：只能看到和管理自己创建的班级和学员，不能看到其他导师的数据
- **学员**：只能看到自己的成长值、礼品和订单，不能看到其他学员的数据
- **班级助理**：只能看到被授权班级的数据，不能看到其他班级的数据

### 7.2 班级开放/关闭对功能的限制

- **开放**：学员可以绑定班级，导师可以添加新学员
- **关闭**：学员不能绑定班级，导师仍可以管理现有学员和数据

### 7.3 库存为0时的兑换限制

- 库存为0时，学员不能兑换该礼品
- 系统会在礼品详情页显示"库存不足"

### 7.4 导师可操作、学员不可操作的边界

- **导师**：创建班级、管理学员、记录成长值、审核兑换、核销订单
- **学员**：绑定班级、查看成长值、兑换礼品、查看订单

### 7.5 超级管理员对模块的开关控制

- 超级管理员可以启用/禁用系统模块
- 可以设置系统参数，如成长值规则、兑换规则等

### 7.6 权限配置逻辑

- **权限验证**：系统通过`require_role`函数验证用户是否拥有相应的角色权限
- **角色依赖**：系统定义了常用的角色依赖，如`require_admin`（超级管理员和管理员）、`require_teacher`（超级管理员、管理员、导师和班级助理）、`require_student`（学员）
- **空值检查**：权限验证会检查用户角色是否存在，避免空指针异常
- **权限继承**：子角色继承父角色的权限，可在此基础上进行调整
- **动态路由**：根据用户权限动态生成前端路由菜单
- **角色管理**：管理员和超级管理员可以在前台建立自定义角色，并为其分配权限

## 9. 关键逻辑判断规则

### 9.1 重名校验、同音姓名匹配

- 系统会对同一班级内的学员姓名进行去重
- 支持同音姓名的模糊匹配，避免重复添加

### 9.2 积分扣减、库存扣减、配额判断

- 兑换礼品时，系统会先检查学员积分是否足够
- 检查礼品库存是否足够
- 扣减积分和库存是原子操作，确保数据一致性

### 9.3 兑换后能否取消、谁能取消

- 待审核状态的订单，学员可以取消
- 已审核状态的订单，只有导师可以取消
- 已核销的订单，不能取消

### 9.4 学员切换不同导师/班级的逻辑

- 学员可以绑定多个班级，但只能有一个当前班级
- 切换班级时，需要导师审核通过
- 历史数据会保留在原班级

### 7.5 排行榜开放/关闭逻辑

- 导师可以设置排行榜的开放/关闭状态
- 关闭后，学员不能查看排行榜
- 排行榜数据会实时更新

### 7.6 排行榜显示范围和过滤条件

- **导师**：查看自己创建的班级的排行榜
- **班级助理**：查看被授权班级的排行榜
- **学员绑定状态过滤**：默认显示 `APPROVED`（已绑定）和 `NONE`（导师导入但未绑定）状态的学员
- **成长值过滤**：只显示总成长值不等于 0 的学员，大于0和小于0的都显示
- **排序规则**：按总成长值降序排列

### 7.7 班级助理权限说明

**助理身份特点**：
- 助理身份只能通过导师授权获得，不能自主注册
- 助理可以查看被授权班级的成长记录、排行榜、成长日志等数据
- 助理不能添加/编辑/删除学员，不能管理礼品和订单

**助理访问限制**：
- 只能访问被授权的班级数据
- 排行榜 API 需要验证班级助理权限
- 成长记录 API 需要验证班级助理权限

**助理绑定班级流程**：
- 助理点击"绑定班级" → 系统清除当前 token → 跳转到登录页 → 用户用学员身份登录 → 绑定班级
- 这是因为绑定班级需要学员身份，助理需要切换为学员身份才能操作

### 9.6 修改库存时锁定礼品防止并发错误

- 修改礼品库存时，系统会对礼品进行锁定
- 确保并发操作时的数据一致性
- 操作完成后解锁

### 9.7 数据变更联动关系

#### 9.7.1 学员解绑逻辑

- **未注册学员（仅导师导入）**：解绑时标记 `is_deleted = 1`，并清空 `student_no_in_class` 字段，释放学号占用，允许班级再次添加相同学号的新学员
- **已注册学员**：解绑时标记 `is_deleted = 1`，但不清空 `student_no_in_class` 字段，避免账号数据混乱
- **学号唯一性校验**：班级内学号唯一性校验仅针对 `is_deleted = 0` 的有效学员，已解绑学员不参与唯一性判断
- **历史记录保留**：解绑后学员在正常列表、统计、排行、成长值、兑换中完全不显示，仅在「历史学员」页面可查看历史记录

#### 9.7.2 历史学员功能

- **历史学员页面**：显示所有已解绑的学员，包括学号、姓名、班级、解绑时间、解绑原因等信息
- **搜索和分页**：支持按学员姓名搜索，分页展示（每页20条）
- **历史记录**：在历史学员页面可以查看学员的完整历史记录，包括成长值、标签等信息

#### 9.7.3 操作日志详情

- **操作前数据**：显示操作前的学员数据，格式化为可读的JSON格式
- **操作后数据**：显示操作后的学员数据，格式化为可读的JSON格式
- **日志记录**：只在数据实际发生变化时记录操作日志，避免不必要的日志记录
- **成长值记录变更**：
  - 添加/修改/删除成长值记录时，系统会自动清除相关排行榜缓存
  - 缓存键模式：`leaderboard:*:{teacher_id}:*`、`leaderboard:all:{teacher_id}:*`、`leaderboard:classes:{teacher_id}`
  - 确保排行榜数据实时更新
- **学员管理操作**：
  - 添加/批量添加/解绑/恢复/更新学员信息/更新学员标签时，系统会清除相关缓存
  - 缓存清除模式：`students:teacher:{teacher_id}:*`
  - 确保学员列表和排行榜数据同步更新
- **成长值计算联动**：
  - 总成长值：只包括通过成长管理增减的记录，不包括兑换记录
  - 可用成长值：包括所有成长记录的增减，包括兑换记录的扣除和删除操作的负记录
  - 排行榜同时显示总成长值和可用成长值
- **兑换操作联动**：
  - 学员兑换礼品时，系统会扣减可用成长值
  - 生成兑换记录，操作类型为"兑换"
  - 同时更新礼品库存
  - 清除相关排行榜缓存
- **删除操作处理**：
  - 删除成长值记录时，系统会添加一条负的成长值记录来表示删除
  - 操作类型为"删除"，记录数值为删除记录的负值
  - 保持操作可溯源，不直接删除原记录
  - 清除相关排行榜缓存

## 8. 成长记录性能优化设计

### 8.1 问题分析

**成长记录显示慢的根本原因**：
- **N+1 查询问题**：获取排行榜时，每个学员都需要单独查询成长值，导致数据库查询次数过多
- **重复计算**：每次获取成长值都需要重新计算，没有缓存机制
- **数据库压力**：频繁的数据库查询导致系统负载过高

### 8.2 优化方案

#### 8.2.1 数据库查询优化

- **批量查询**：实现 `_get_batch_growth_scores` 方法，批量获取所有学员的成长值，减少数据库查询次数
- **减少 N+1 查询**：在 `get_class_leaderboard` 中实现批量获取活跃学员和班级成长值的查询
- **合理使用索引**：为高频查询字段添加索引，如 `growth.class_student_id`、`growth.created_at` 等

#### 8.2.2 缓存机制实现

**缓存策略**：
- **高频数据缓存**：对排行榜、成长值、成长值流水等高频查询数据进行缓存
- **缓存键设计**：使用结构化的缓存键，包含必要的参数信息
- **过期时间**：根据数据更新频率设置不同的过期时间（2-5分钟）
- **主动更新**：数据变更时主动清除相关缓存，确保数据一致性

**缓存实现**：
- `get_leaderboard`：缓存有效期 5 分钟
- `get_growth_score`：缓存有效期 5 分钟
- `get_growth_logs`：缓存有效期 2 分钟
- `get_growth_history`：缓存有效期 2 分钟
- `get_class_leaderboard`：缓存有效期 5 分钟

#### 8.2.3 性能监控

- **时间统计**：在各个方法中添加时间统计，记录方法执行时间
- **日志记录**：添加缓存命中日志，便于性能分析
- **监控指标**：记录缓存命中率、平均响应时间等指标

### 8.3 预期效果

- **首次加载速度提升**：通过批量查询，减少数据库交互次数
- **后续加载速度显著提升**：由于缓存的存在，响应时间将从毫秒级降至微秒级
- **系统负载降低**：减少了数据库查询次数，减轻了系统负担
- **用户体验改善**：成长记录页面的加载速度将明显加快，尤其是在频繁访问时
